🔧 Heuristic Search, Pathfinding & State Spaces

1. Follow the graph where edge weights show step costs and nodes show heuristic values. Find the most cost-effective path from start to goal using appropriate algorithms.

Technique: A* Search Algorithm

  • f(n) = g(n) + h(n)
    where g(n) = cost from start to node n, h(n) = heuristic estimate from n to goal

2. Explain Alpha-Beta Pruning. Define alpha and beta, and solve an example. Also solve a cryptarithmetic puzzle like SEND+MORE=MONEY.

A. Alpha-Beta Pruning in Minimax:

  • Alpha (α): Best (highest) value that MAX can guarantee so far.

  • Beta (β): Best (lowest) value that MIN can guarantee so far.

  • Prune: Stop evaluating when α ≥ β

Game Tree Example:

markdown
         MAX
        /   \
      MIN   MIN
     / \    / \
   3   5  2   9

Minimax Value:

  • Left MIN = min(3,5) = 3

  • Right MIN = min(2,9) = 2

  • MAX = max(3,2) = 3

Using Alpha-Beta:

  1. MAX looks at MIN(3,...) → α=3

  2. If MIN has 2 < α, prune rest
    🔪 Prune the 9

Constraints:

  • M = 1 (carry from 4-digit + 4-digit = 5-digit)

3. Solve the 8-puzzle problem from a given initial state to goal using Manhattan heuristic.

Manhattan Heuristic (h):
Sum of horizontal + vertical distances of each tile from its goal position.

Initial:

1 2 3
4 _ 6
7 5 8

Goal:

1 2 3
4 5 6
7 8 _

Misplaced Tiles: 5, 8 → Not enough
Manhattan Distance:

  • 5: From (2,1) → (1,1) → h = 1

  • 8: From (2,2) → (2,1) → h = 1
    Total h = 2

Use A*:

  • Move empty tile (_), generate successors

  • Track g(n) (steps so far), h(n) (heuristic), f(n)

Continue expanding the lowest f(n) node until goal state reached.

4. Give PDDL descriptions for planning problems such as air cargo or block stacking.

Example: Block Stacking

Blocks: A, B, C
Initial:

  • On(A, B), OnTable(B), OnTable(C), Clear(A), Clear(C)
    Goal:

  • On(C, A), On(A, B)

PDDL Components:

lisp
(define (problem blockstack)
  (:domain blocks)
  (:objects A B C)
  (:init
    (on A B)
    (ontable B)
    (ontable C)
    (clear A)
    (clear C)
  )
  (:goal
    (and
      (on C A)
      (on A B)
    )
  )
)

✅ This describes initial and goal states.
You also need :domain with actions like stack, unstack, pickup, putdown in PDDL.
