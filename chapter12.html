Q6. Define admissible and consistent heuristics.

Admissible Heuristic: A heuristic h(n) is admissible if it never overestimates the cost to reach the goal from node n. Formally:

h(n) ‚â§ h*(n) for all n, where h*(n) is the true cost to reach the goal.

Consistent (Monotonic) Heuristic: A heuristic h(n) is consistent if:

h(n) ‚â§ c(n, a, n') + h(n') for every node n and successor n', where c(n, a, n') is the cost of action a.

  • Every consistent heuristic is also admissible.

Example (8-Puzzle):

  • h‚ÇÅ(n) = number of misplaced tiles ‚Üí admissible
  • h‚ÇÇ(n) = total Manhattan distance ‚Üí admissible and consistent

Impact on A*:

  • Admissibility ensures A* is optimal.
  • Consistency allows A* to avoid re-visiting nodes.

Q8. What is arc consistency? Explain AC-3.

Arc Consistency: An arc X ‚Üí Y is consistent if for every value in the domain of X, there is some allowed value in Y satisfying the constraint.

AC-3 Algorithm: A popular algorithm to enforce arc consistency on all arcs in a CSP.

Steps:

  1. Initialize queue with all arcs
  2. While queue not empty:
    • Remove (Xi, Xj)
    • If domain of Xi is revised, add all arcs (Xk, Xi) back

Pseudocode:

function AC-3(csp):
    queue ‚Üê all arcs in csp
    while queue is not empty do:
        (Xi, Xj) ‚Üê REMOVE-FIRST(queue)
        if REVISE(csp, Xi, Xj) then
            if domain of Xi is empty then return false
            for each Xk in neighbors[Xi] do
                add (Xk, Xi) to queue
    return true

Example: X ‚àà {1,2,3}, Y ‚àà {2,3}, with constraint X < Y ‚Üí Remove 3 from X

Q14. What is PDDL? Write PDDL for a pickup action.

PDDL (Planning Domain Definition Language): A standard language for expressing planning domains and problems.

Structure:

)

This defines that the agent can pick up an object if it's at a location and the agent‚Äôs hand is empty.
